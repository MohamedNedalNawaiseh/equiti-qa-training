Swagger + Postman Integration

Import API definitions and master API testing with Postman

Introduction

Postman is a powerful API testing tool that allows you to design, build, test, and document APIs. Combined with Swagger/OpenAPI specifications, you can quickly generate complete test suites.

Why Use Postman?

  • Import Swagger: Generate collections from API specs automatically
  • Organize APIs: Group related requests into collections
  • Environment Variables: Test across dev, staging, prod easily
  • Test Automation: Write pre-request and test scripts
  • Mock Servers: Simulate APIs before they're built
  • CI/CD Integration: Run collections in pipelines

Collections

Collections are containers that organize related API requests. They can include documentation, variables, and auth settings.

Collection Structure

Collection (Folder-based)
├── Auth
│   ├── POST /auth/login
│   └── POST /auth/refresh
├── Users
│   ├── GET /users
│   ├── POST /users
│   ├── GET /users/{id}
│   ├── PUT /users/{id}
│   └── DELETE /users/{id}
└── Documentation
    └── README.md (collection description)

Collection Settings

  • Variables: Collection-level variables
  • Auth: Inherited auth for all requests
  • Pre-request Script: Run before each request
  • Test Script: Run after each request
  • Documentation: Collection description and notes

Variables

Postman supports multiple variable scopes: Global, Collection, Environment, Data, and Local.

Variable Scopes (Priority: High to Low)

1. Global Variables
   Available everywhere, set via: pm.globals.set("var", "value")
   Access via: {{globalVar}} or pm.globals.get("globalVar")

2. Collection Variables
   Available in collection, set via: pm.collectionVariables.set("var", "value")
   Access via: {{collectionVar}}

3. Environment Variables
   Available in active environment, set via: pm.environment.set("var", "value")
   Access via: {{envVar}}

4. Data Variables
   From CSV/JSON data files during collection runs

5. Local Variables
   Temporary, set via: pm.variables.set("var", "value")
   Only available in current request/script

Variable Syntax

// Double curly braces - auto-resolved in:
- URL
- Headers
- Body (JSON/XML raw)
- Pre-request Script (some cases)

// pm.variables.get() - for scripts
const baseUrl = pm.variables.get("baseUrl");

// Dynamic variables (built-in)
{{$guid}}     // Generate UUID
{{$timestamp}} // Current timestamp
{{$randomInt}} // Random integer

Setting Up Auth

// Method 1: At collection level (inherited by all requests)
Collection → Authorization → Bearer Token → {{token}}

// Method 2: At request level (overrides collection)
Request → Headers → Authorization: Bearer {{token}}

// Method 3: Via pre-request script (dynamic)
pm.request.headers.add({
    key: 'Authorization',
    value: 'Bearer ' + pm.variables.get("token")
});

OAuth 2.0 Flow

// OAuth 2.0 Configuration in Postman
Auth Type: OAuth 2.0

Configuration:
- Callback URL: http://localhost:3000/callback
- Auth URL: https://auth.example.com/authorize
- Access Token URL: https://auth.example.com/token
- Client ID: your-client-id
- Client Secret: your-client-secret
- Scope: read write
- State: random-state-string

Token Generation:
1. Click "Get New Access Token"
2. Postman opens auth URL in browser
3. User logs in and grants permission
4. Postman receives callback with auth code
5. Postman exchanges code for access token
6. Token saved and can be used in requests

Test Scripts

Postman allows you to write JavaScript to validate responses, set variables, and create dynamic workflows.

Test Script Example

// Tests tab - runs after request completes
pm.test("Status is 200", function() {
    pm.response.to.have.status(200);
});

pm.test("Response has users array", function() {
    const response = pm.response.json();
    pm.expect(response).to.have.property('users');
    pm.expect(response.users).to.be.an('array');
});

pm.test("First user has required fields", function() {
    const user = pm.response.json().users[0];
    pm.expect(user).to.have.property('id');
    pm.expect(user).to.have.property('email');
    pm.expect(user).to.have.property('name');
});

pm.test("Response time is acceptable", function() {
    pm.expect(pm.response.responseTime).to.be.below(500);
});

Pre-request Script

// Pre-request Script - runs before request
// Example: Generate timestamp and add to header
const timestamp = Date.now();
pm.request.headers.add({
    key: 'X-Request-Time',
    value: timestamp
});

// Example: Load test data
const testUser = {
    email: 'test' + Date.now() + '@example.com',
    name: 'Test User ' + Math.random().toString(36).substr(2, 9)
};
pm.variables.set("testEmail", testUser.email);
pm.variables.set("testName", testUser.name);

// Example: Set dynamic auth token
const token = pm.globals.get("authToken");
if (!token || isTokenExpired(token)) {
    // Refresh token logic
}

Common Assertions

pm.expect(pm.response.status).to.equal(200);
pm.expect(pm.response.status).to.be.oneOf([200, 201]);
pm.expect(response.name).to.include('John');
pm.expect(response).to.have.property('users').with.lengthOf(10);
pm.expect(response).to.have.all.keys('id', 'email', 'name');
pm.expect(response.email).to.be.an('email');

Chaining Requests

Chain requests by extracting data from one response and using it in subsequent requests.

Login → Get Profile Example

// REQUEST 1: POST /auth/login
// Test Script:
pm.test("Login successful", function() {
    pm.response.to.have.status(200);
});

const response = pm.response.json();
pm.test("Token received", function() {
    pm.expect(response).to.have.property('token');
});

// Save token for next request
pm.collectionVariables.set("authToken", response.token);
pm.collectionVariables.set("userId", response.user.id);

// REQUEST 2: GET /users/me (uses {{authToken}} in auth)
// No test needed, uses inherited token

// REQUEST 3: GET /users/{{userId}}/orders
// Uses the userId from login response

Dynamic Data in Body

// POST /users - Create user with dynamic data
{
    "name": "{{userName}}",        // Set in pre-request script
    "email": "{{userEmail}}",      // Set in pre-request script
    "createdAt": "{{timestamp}}"   // Generated dynamically
}

// Pre-request script to set dynamic values
const timestamp = new Date().toISOString();
const randomName = 'User_' + Math.random().toString(36).substr(2, 8);

pm.variables.set("timestamp", timestamp);
pm.variables.set("userName", randomName);
pm.variables.set("userEmail", randomName + '@test.com');
